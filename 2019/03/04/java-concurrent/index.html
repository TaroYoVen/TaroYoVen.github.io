并发类的知识不管在哪一门编程语言中，都是属于进阶类知识，属于编程语言中高级部分。主要是因为它涉及到了很多底层的知识，如果你对操作系统相关的知识一无所知，那去理解一些原理会费些力气。

编写正确的并发程序是一件极困难的事情，并发程序的Bug往往会诡异地出现，然后又诡异地消失，很难重现，也很难追踪。只有透过现象看本质，才能追本溯源，深入分析出这些Bug的源头。

这些年，我们的cpu，内存，I/O设备都在不断的迭代更新，不断地朝着更快的方向努力着。而并发程序的核心矛盾是也一直存在，就是这三者的速度差异。cup远大于内存，内存又远大于I/O设备。
